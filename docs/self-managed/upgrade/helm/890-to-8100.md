Upgrade a Helm-managed Camunda 8 Self-Managed deployment from version 8.9 to 8.10.

:::info Upgrade procedure
All Camunda 8 upgrades must follow the required upgrade procedure: upgrade one minor version at a time and do not skip minors. For best stability and fix coverage, use the latest available patch in each minor before and after the minor upgrade.

See [version compatibility checks](../../components/orchestration-cluster/core-settings/concepts/version-compatibility.md#required-upgrade-procedure).
:::

:::warning Helm v4 CLI is required
Camunda 8.10 (chart 15.x) requires the Helm v4 CLI. Camunda 8.9 (chart 14.x) is the last minor that supports the Helm v3 CLI. Switch your tooling to the Helm v4 CLI before you run `helm upgrade`. No release-state migration is required when switching CLIs. See [Move from the Helm v3 CLI to v4](/self-managed/deployment/helm/operational-tasks/moving-helm-v3-to-v4.md).
:::

## Prerequisites

Before upgrading, ensure you have met the following prerequisites:

| Prerequisite                | Description                                                                                                                                                                                                                                                                                        |
| :-------------------------- | :------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Confirm `8.9.x` baseline    | Confirm your deployment is running an `8.9.x` version before upgrading to 8.10. If you are upgrading from a version earlier than 8.9, see [Upgrading from an earlier version](/self-managed/upgrade/index.md#upgrading-from-an-earlier-version). For best stability, use the latest `8.9.x` patch. |
| Confirm upgrade eligibility | Review [Prepare for upgrade](/self-managed/upgrade/prepare-for-upgrade.md) to confirm upgrade eligibility and complete any required pre-upgrade actions for 8.10.                                                                                                                                  |
| Switch to the Helm v4 CLI   | Chart 15.x requires the Helm v4 CLI. See [Move from the Helm v3 CLI to v4](/self-managed/deployment/helm/operational-tasks/moving-helm-v3-to-v4.md).                                                                                                                                               |
| Confirm Helm chart support  | Review the Camunda Helm chart [version matrix](https://helm.camunda.io/camunda-platform/version-matrix/) and confirm the chart version deploying 8.10 is supported for your Kubernetes version.                                                                                                    |
| Backups                     | Create and verify backups for your Camunda data. See [Backup and restore](../../../self-managed/operational-guides/backup-restore/backup-and-restore.md).                                                                                                                                          |
| Check your `values.yaml`    | Review your existing `values.yaml` for removed subchart keys and for the Console and Web Modeler consolidation described below.                                                                                                                                                                    |
| Test upgrade                | Test the upgrade in a non-production environment using a copy of your production configuration.                                                                                                                                                                                                    |

## Configuration changes

### Camunda Hub: Console and Web Modeler consolidation

In 8.10, Console and Web Modeler are consolidated into **Camunda Hub**. Web Modeler continues to run as a deployment; Console is no longer a standalone component and is now provided as an in-Modeler feature within Camunda Hub.

The Helm upgrade is **zero-touch at the values level**: the legacy top-level `console.*` and `webModeler.*` keys continue to work unchanged, so an existing 8.9 values file deploys the same components in 8.10. When either legacy key is set, the chart emits a deprecation warning on `helm upgrade`.

To adopt the new configuration:

- Set `camundaHub.enabled: true` to deploy Camunda Hub (equivalent to enabling both Web Modeler and the in-Modeler Console).
- Move Web Modeler overrides under `camundaHub.webModeler.*` and Console overrides under `camundaHub.console.*`. Values placed under `camundaHub.*` take precedence over the legacy top-level keys.

```yaml
# Legacy 8.9 configuration (still works in 8.10, emits a deprecation warning):
console:
  enabled: true
webModeler:
  enabled: true

# Recommended 8.10 configuration:
camundaHub:
  enabled: true
```

:::note Console application configuration
The consolidation removes the standalone Console application configuration. If you configured Console through application properties (`camunda.console.*`) or `CAMUNDA_CONSOLE_*` environment variables — rather than through the Helm `console.*` values — you must migrate that configuration. Console cluster-management settings are ported to Camunda Hub. See the [component upgrade guide](../components/890-to-8100.md#camunda-hub).
:::

### Removed bundled subchart keys

The bundled Bitnami database subcharts are removed in 8.10. If your 8.9 values file still sets any of the following keys, `helm upgrade` fails until you remove them and migrate to an externally managed service:

- `identityKeycloak`
- `identityPostgresql`
- `webModelerPostgresql`

Provide an external PostgreSQL instance for Web Modeler under `webModeler.restapi.externalDatabase` (or `camundaHub.webModeler.restapi.externalDatabase`), and an external Keycloak/PostgreSQL for Identity as applicable.

### Other upgrade notes

For additional 8.10 Helm changes — including the Elasticsearch subchart no longer being enabled by default, the required `orchestration.data.secondaryStorage.type` setting, and the Web Modeler `restapi` ephemeral volume — see the [Helm upgrade notes](./index.md#upgrade-notes) and the [component upgrade from 8.9 to 8.10](../components/890-to-8100.md).

## Run the upgrade

After updating your values file, run the Helm upgrade with the Helm v4 CLI:

```bash
helm repo update
helm upgrade camunda camunda/camunda-platform --version <chart-15.x-version> \
  --namespace <namespace> \
  -f values.yaml
```

Use the Helm chart [version matrix](https://helm.camunda.io/camunda-platform/version-matrix/) to identify the chart 15.x version that deploys your target 8.10 patch.
