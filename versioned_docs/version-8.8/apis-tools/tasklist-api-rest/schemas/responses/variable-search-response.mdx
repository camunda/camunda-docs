VariableSearchResponse - represents a variable in a search response. It contains information about the variable's ID, name, value, truncation status, and a preview of the value. The `value` field may be `null` if the value has been truncated due to exceeding a length limit (8191 characters). The `previewValue` field provides a shorter preview of the value for display purposes.

### Fields

#### [<code style={{ fontWeight: 'normal' }}>VariableSearchResponse.<b>id</b></code>](#)<Bullet />`string` <Badge class="secondary" text="non-null"/>

> id of the variable

#### [<code style={{ fontWeight: 'normal' }}>VariableSearchResponse.<b>name</b></code>](#)<Bullet />`string` <Badge class="secondary" text="non-null"/>

> variable name

#### [<code style={{ fontWeight: 'normal' }}>VariableSearchResponse.<b>value</b></code>](#)<Bullet />`string` <Badge class="secondary" text="non-null"/>

> represents the value of the variable. If `isValueTruncated` is `true` and `alwaysReturnFullValue` in the search request is not set to `true`, the value will be `null`.

#### [<code style={{ fontWeight: 'normal' }}>VariableSearchResponse.<b>previewValue</b></code>](#)<Bullet />`string` <Badge class="secondary" text="non-null"/>

> preview of the variable value. It is limited to a length of 8191 characters.

#### [<code style={{ fontWeight: 'normal' }}>VariableSearchResponse.<b>isValueTruncated</b></code>](#)<Bullet />`boolean` <Badge class="secondary" text="non-null"/>

> indicates, whether `previewValue` contains truncated value or full value

#### [<code style={{ fontWeight: 'normal' }}>VariableSearchResponse.<b>draft</b></code>](#)<Bullet />`object` <Badge class="secondary" text="non-null"/>

> represents draft value for variable
